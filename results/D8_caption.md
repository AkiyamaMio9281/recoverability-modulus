# D8 — SFBD-OMNI 算法动力学:Ψ 控制 clean 空间收敛 caption

文件:`D8_omni_dynamics.png`(闭式高斯 OMNI,d=5,σ=0.3,5 seeds,50 迭代,κ∈{1,5,25,125,625})。

**做了什么(与 D2/D3/D5 的关键区别)**:前面用的是**理想估计量**;这里跑**真实 SFBD-OMNI 交替
最小化**(闭式,`src/recmod/omni.py`,推导见 `notes/M6`),在算法**自己的迭代**里观察。

**三个面板**:
- **A. 腐蚀空间**:`KL(q‖T_r p^k)` 对**所有** κ 都收敛——算法"在腐蚀空间看起来成功"。
- **B. clean 空间**:`W2(p^k, p_data)` 随 κ=Ψ 增大而**停滞**在越来越高的水平。
- **C. gap**:末迭代,clean `W2`(红)随 κ 升;corrupted `KL`(蓝)始终很小。

**最强一击**:κ=625 时腐蚀 KL=3.9e-6(看起来完全收敛!),clean W2=0.53(彻底失败)。
**算法在最病态时、腐蚀空间看起来最"收敛",clean 空间却最离谱。** 这就是 OMNI 自己点出的
*Identifiability ≠ Recoverability*——现在被 Ψ 在它**自己的轨迹**上定量展示,而非定性吐槽。

**意义**:D8 把 D6(clean 误差 ≈ Ψ×√腐蚀差)从"定理骨架"升级为"**对着真算法验证过**";
直接回应"这跟真实方法有什么关系?"——它就是在跑 Yu 自己的算法。CPU、闭式、40 tests 全绿。

**边界**:高斯-线性、γ=1、λ=0、injective;非线性/非高斯、λ>0 非单射 + h† 是下一步。
corrupted-KL 在大 κ 处随 κ 非单调,值得与 Yu 讨论;但 gap 结论稳健。
